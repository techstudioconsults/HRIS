# Settings Module — Glossary

_Domain terms and abbreviations used across all Settings module documentation._

## Terms

| Term                      | Definition                                                                                               |
| ------------------------- | -------------------------------------------------------------------------------------------------------- |
| **Organisation Settings** | Configuration values that apply to the entire organisation and all its employees.                        |
| **Account Settings**      | The tab covering identity-level org info: name, logo, contact details, registration number.              |
| **Payroll Settings**      | The tab covering payroll processing configuration: pay cycle, currency, deduction rules.                 |
| **Security Settings**     | The tab covering access security policies: 2FA enforcement, session timeout, password policy.            |
| **HR Settings**           | The tab covering human resources operational defaults: working hours, probation period, leave carryover. |
| **Notification Settings** | The tab covering which events trigger email or in-app notifications across the org.                      |
| **Roles Management**      | The tab for creating and configuring custom RBAC roles and their permission assignments.                 |
| **Pay Cycle**             | The interval at which employees are paid (e.g., weekly, bi-weekly, monthly).                             |
| **Deduction**             | A reduction from gross pay (e.g., tax, pension contribution, health insurance).                          |
| **2FA**                   | Two-Factor Authentication — a second verification step beyond username/password.                         |
| **Session Timeout**       | The duration of inactivity after which a user is automatically signed out.                               |
| **Probation Period**      | The initial employment period during which different leave and HR rules may apply.                       |
| **Leave Carryover**       | The policy governing how unused annual leave days are transferred to the next leave year.                |
| **Custom Role**           | An admin-created role with a user-defined name and selectable permission set.                            |
| **System Role**           | A hard-coded role (Super Admin, HR Manager, etc.) that cannot be modified via the UI.                    |
| **Permission Scope**      | A discrete access grant string (e.g., `admin:resources:write`) assigned to a role.                       |

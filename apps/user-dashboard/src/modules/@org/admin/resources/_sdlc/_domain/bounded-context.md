# Resources Module — Bounded Context

_Defines the domain boundary for company document and file management._

## Context Name: Resource Management

### Responsibilities

- Owning the lifecycle of `ResourceFolder` and `ResourceFile` aggregates.
- Enforcing folder hierarchy rules (parent-child relationships, name uniqueness within scope).
- Managing file storage references (storage keys are internal; download URLs are issued on request).
- Enforcing organisation-scoped access (a folder or file belongs to exactly one organisation).

### What This Context Does NOT Own

- Employee records (belongs to the Employee context).
- Payroll documents generated by the payroll engine (payslips are owned by the Payroll context; they may be stored here as read-only archives).
- User authentication and session management (belongs to the Auth context).

### Integration Points

| Upstream Context | Integration           | Description                                                            |
| ---------------- | --------------------- | ---------------------------------------------------------------------- |
| Auth             | JWT claims            | Organisation ID and user ID are read from the JWT to scope all queries |
| Employee         | Employee ID reference | `createdBy` / `updatedBy` fields reference employee user IDs           |

| Downstream Context | Integration   | Description                                                                         |
| ------------------ | ------------- | ----------------------------------------------------------------------------------- |
| Notifications      | Domain events | `resource.file.uploaded` may trigger an in-app notification to subscribed employees |
| Audit Log          | Domain events | All mutations are forwarded to the audit log service                                |

### Ubiquitous Language

Within this context, use these terms precisely:

- **Folder** — not "directory", "category", or "bucket".
- **File** — not "document", "attachment", or "asset" (except in UI copy where "document" is used for clarity).
- **Upload** — not "attach" or "import".
- **Organisation** — not "company" or "tenant" in code; "organisation" is the canonical term.
